SONIC VLA 鲁棒性方案 v2
Make most realistic deploy poses prompt-triggerable — revised after codex + glm dual review

TL;DR(v2,被两个评审推翻重写). v1 想"重训策略让随机初始姿都能触发", 被 codex(gpt-5.5)+ glm-5.2 一致打回:那是在用重训补偿一个复位 bug,且判据/数据策略多处错误。

新顺序: Phase 0 零重训先验 先做两件事 —— ①WBC 隔离 ablation(冻策略喂 GT token,看 WBC 跟不跟得住;跟不住=重训策略白搭)+ ②把部署复位姿锁回训练标称姿(0.54rad 偏差是 reset/init 问题,不是策略问题)。多半到这就 close ticket。
Phase 1 真要重训 才上精简配方:输入 proprio 噪声增广 + LoRA(冻骨干)+ 零初始化 history adapter。 丢弃 RSI 切片(扩错维度)、丢弃 DART/DAgger(off-tube 无 ground-truth token,不可行)。
判据:不用 drift>1.5(摔也高 drift=假阳性);用任务级闭环判据(kick 脚高/jump 质心升/walk 位移)+ 轨迹方向对齐。

1. 问题回顾(诊断已坐实)Diagnosis recap

多轮单变量受控实验(checkpoint-8000,本机 4090)结论:闭环里 kick/jump/walk 站立不动,squat/dance/lunge/macarena 正常。 逐一证伪:动量说、模型预测能力、重力 OOD、图像、单关节组 bug。证实:prompt 有效选动作; 真因 = 单帧无历史策略对脆弱动作"训练轨迹细管子"零容差——部署复位姿整体偏 ~0.54rad(分散全关节、无单点 bug)+ 闭环 WBC 跟踪持续漂移 → 累计 OOD → 模型把 40 步预测塌成"保持当前姿"的冻结 chunk。完整证据见 doc/sonic_vla_closeloop_validation.html。

3. 目标的可量化定义(修订)Measurable goal, revised

不再用 drift,也不再用"随机姿"。目标 = 从现实部署会遇到的初始姿分布,prompt 触发任务级成功。

要素v2 定义
初始姿分布三桶分别报告:标称姿(reset 名义站姿)/ 轻扰 / 重扰;扰动从实测 reset+WBC-drift 协方差采样(非独立均匀 ±0.15,那会造非自然姿),钉死 σ 与协议
触发成功(任务级,主门)闭环 WBC rollout 同时满足:安全(不摔/root 高度/tilt)+ 触发延迟(prompt 后 ≤1s 进入)+ 动作专属事件(kick=摆动脚最大高度&支撑脚保持;jump=质心升+落地;walk=连续交替触地+前向位移;squat=骨盆下沉再起)+ 选择性(错 prompt 不触发该判据)+ 持续/完成
轨迹相似(辅)预测 chunk 对该动作 GT 做 phase-invariant DTW / 关节轨迹 PCA 方向对齐;drift 仅作粗筛冻结检测,绝不单独当门
样本量每动作 ≥100 初始姿(drift 粗筛可大,任务级闭环子集 N≈5–10/动作)
分级目标squat/dance/lunge/macarena 维持;kick/jump 较硬;walk(极限环)最硬,接受 <80%。区分"策略不行"vs"物理不可达"

4. Phase 0 零重训先验证(最高优先)Zero-retrain first

两评审一致:动工重训前必须先做这两步,否则可能全程在修错的那一段。多半到这里问题就解决/边界就清楚了。

0.1 WBC 隔离 ablation —— 地基

冻当前策略,从同一复位姿喂 GT kick token(开环 dump),看 WBC 能不能跟出 kick。

  • 能 → WBC 没问题,瓶颈在策略生成,Phase 0.2 / Phase 1 有意义。
  • 不能 → WBC 才是天花板,重训策略白干,应改 WBC / reset controller 或限制目标 basin。
脚手架:BOOTSTRAP_STEPS=<full> bash scripts/gear_sonic_live_demo.sh kick(全程喂 dump token)对照纯 live。

0.2 复位锁回训练分布 + 部署旋钮扫

0.54rad 是 reset/init 偏差。固定现 ckpt,把部署起姿锁回训练标称姿(把偏差压到 <0.1rad),再扫这些不重训旋钮:

旋钮做法
精确复位env reset 时把机器人关节设成数据集首帧姿(已证:首帧喂 GT proprio → kick drift 0.31→2.76、dpose→3.12 真踢)
bootstrap 长度0 / 20 / 40 / 80 扫,找"能起飞且最少回放"的点(已证 80→live 可持续)
execution horizon / RTC每次 query 消费的 token 步数(reactivity vs 平滑);real-time chunking
若 0.2 让 kick/jump 起得来 → 整个重训计划可能不必要,close ticket。walk(极限环)即便复位正确也可能仍需 Phase 1。

5. Phase 1 重训配方(精简,仅当 Phase 0 不够)Slim retrain recipe

主 A. 输入 proprio 噪声增广 Input-noise augmentation = the feasible DART kernel

为什么: 0.54 偏差活在轨迹"正交方向";给训练样本的 proprio 输入加噪声(标签不变)= 把"管子周围一圈"也变成训练分布 = 扩 basin。这是 DART 的可行内核,不需要任何新标注,也不需要 RSI 切片。

# 训练时对 observation.state 加噪(从实测 reset/drift 协方差采样,非独立均匀):
state_noisy = state + sample_from(Σ_deploy)     # σ≈0.05–0.1rad 量级,含少量尾部
# 标签(未来40 token)不变。history dropout 同理(见 B)。

主 B. LoRA(冻骨干)+ 零初始化 history adapter LoRA + zero-init history

  • LoRA on DiT head(rank 8–16),冻 VLM + 冻 DiT 主干: 7-ep 小数据标配,参数砍两个数量级、自带正则。LoRA 的 B 矩阵天然零初始化 → 热启动不退化。
  • 历史观测改 zero-init residual adapter(不要插值): 若加 delta_indices=[-1,0],-1 帧走独立小 MLP,末层权重置 0 → step-0 数值 ≡ 旧 ckpt,梯度自己慢慢学用历史。episode 首帧 q_{t-1}=q_t + history_valid=0 并在训练模拟。
  • history dropout(p≈0.5): 随机丢 -1 帧,强迫不全靠历史,渐进启用。
  • 先只 state 历史,不加图像历史(改冻 VLM 输入分布、收益不明)。
训练纪律: clean:noisy = 1:1 replay 防遗忘;loss 按动作均衡(kick 仅 165 帧,别让 macarena 1375 帧主导); train/val 不能随机帧切分(同轨迹相邻帧泄漏)→ 按扰动 seed/类型/幅度分层隔离。

P3 后备 C. 观测加速度 / 扩数据

先修 projected_gravity 数据恒零 bug(重训记录真实值);base 线/角速度 + 关节速度比纯历史更直接暴露动力学。每动作多采 3–5 条独立 demo 才能验风格/起姿泛化。优先级低于 A/B。

6. Ablation 计划(否则无法归因)Ablation

  1. Phase 0.2 only(精确复位+bootstrap 扫,不重训)→ 基线,可能已达标。
  2. + A(输入噪声,LoRA)→ 看 basin 触发率增量。
  3. + B(零初始化 history adapter + dropout)→ 增量。
  4. + C(速度/扩数据)→ 仅当 1–3 不够。

每档用第 3 节任务级判据评,且原 4 个 work 动作闭环成功率下降 ≤5 个百分点为硬约束。

7. 风险(评审补全)Risks

风险缓解
修错对象(策略 vs WBC)Phase 0.1 WBC 隔离先验,没验不开工
kick/walk basin 物理不可达(从多数姿得先"恢复到可踢姿"=另一个动作)目标按动作分级,区分策略不行 vs 物理不行
拆掉学到的安全 fallback:冻结可能是 OOD 时的隐式安全行为,强行吐动作 → 自信错误 → 摔安全判据(不摔)优先于触发;监控跌倒率不升
prompt 选择性退化:proprio 方差↑ → prompt 相对方差↓history dropout + 矩阵监控非对角;必要时加 prompt 通道容量
7-ep 小数据 + 可训容量 = 过拟合LoRA + 冻骨干 + 1:1 clean replay + 分层 split
一句话: 先 Phase 0(WBC 隔离 + 精确复位)——多半够;真要重训,输入噪声 + LoRA + 零初始化 history 三件就够,RSI 切片/DART 删掉;判据换任务级闭环。 这与既有 roadmap "先 loss→history→数据" 一致,但顺序前置了"先把复位修对",并把 history 落成零初始化 adapter 这一可执行形态。

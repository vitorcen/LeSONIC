SONIC-VLA 底座替代模型调研
Which open models (besides GR00T) can drive the FSQ-token → SONIC-WBC pipeline?

2026-06-11 · 三方头脑风暴：Claude (Fable 5) + codex (gpt-5.5) + mimo-v2.5-pro，外加两路 web 事实调研（2025–2026 发布、许可证、人形先例逐一核查）。场景与锚点见 sonic_starvla_swap_brainstorm.html §11。

TL;DR — 三方独立分析高度收敛：这个任务的本质不是「操作臂 VLA」，而是 prompt-conditioned 离散 motion-token 生成（视觉只是弱条件）。因此推荐排序为：
🥇 现有 StarVLA CE 路线深化（Qwen 系 VLM + 自研离散 head）——三方一致 Top-1；
🥈 RDT2-VQ（清华，Apache-2.0）——全场唯一原生目标函数就是「CE over 离散动作 token」的大厂级发布，概念完全同构，代价是 7B 在 24G 边缘 + proprio 接口是死代码；
🥉 小型 masked-transformer 从零搭（MoMask / 1xgpt 模板，50–200M）——对 8 clip 极小数据最稳、4090 最轻松、且 "FSQ token + masked 并行解码 + 人形 WBC" 截至 2026-06 无人发表 = 空白生态位。
All three reviewers converge: this is discrete motion-token generation, not arm-VLA. Top-3 = ① extend the proven StarVLA-CE route, ② RDT2-VQ (only major release whose native objective is CE over discrete action tokens), ③ a small from-scratch masked transformer — an unoccupied research niche.

SONIC 的 FSQ 真实配置（gear_sonic/config/actor_critic/universal_token/all_mlp_v1.yaml）： num_fsq_levels: 32 × 32 维 × max_num_tokens: 2 → 64 维 flatten，每维 32 级（值域 [-1, 15/16]，1/16 网格）。 我们 CE head 的 33 bins（k∈[-16,16]）是该网格的覆盖超集，k=+16 是死 bin——无害（softmax 学会不选它），但真实词表是 32 级/维。后续如做 masked 解码，33 = 32 级 + [MASK] 正好是合理 head 宽度。

4. 关键洞察 Key insights

4.1 任务本质重定位 Reframing the task

codex 一针见血：这不是「操作臂 VLA 选型」，是 prompt + proprio/history → 离散 motion-code 序列生成，视觉只是弱条件（ego 相机看地面）。所以「找更好的 VLA」是错误提问；正确提问是「找/搭最好的条件化离散序列生成器」。这解释了为什么所有 stock flow/diffusion head 都塌成模板，也解释了 motion-token LM 路线为什么对题。

4.2 2560-token 问题（工程红线）The 2560-token wall

64 token/步 × 40 步 = 2560 个离散 token。π0-FAST 在 4090 上解码 30–60 个 AR token 要 ~750ms → 逐 token 自回归整 chunk 要 30–60 秒，闭环 2.5Hz 直接不可行。可行解三种，我们现有 CE head 已经天然是第①种：

解法机制先例我们的状态
① 单次并行 per-position head一次前向吐全部 40×78 logitsOpenVLA-OFT（离散变体 90.2% LIBERO）✅ 现状即此——但逐维独立，不建联合分布
② MaskGIT 迭代去掩码10–12 轮置信度重掩码，轮数与网格大小无关MoMask（动作域）、1xgpt（机器人 token 网格）、Discrete Diffusion VLA🎯 最有希望的下一刀：给 head 加上维间/时间联合建模
③ BPE 压缩 token 流FAST .fit() API 在自己数据上训压缩器π0-FAST（Apache-2.0 独立组件）备选；SONIC token 本身已是压缩表示，收益存疑

4.3 空白生态位 The unoccupied niche

领域已收敛到「语言 → 连续 latent → RL tracker」（GR00T 回归 FSQ、LeVERB 用 CVAE latent、Kimodo 绕开 token、FRoM-W1 用自有 VQ）。「对 SONIC 的 64×32 FSQ 网格做 AR/masked 离散建模」截至 2026-06-11 没有任何发表（三个调研线程交叉确认）。我们的 CE 路线 + 下一步 masked 解码正踩在这个空位上——做出来即是可发表的差异化。

4.4 容量甜点与过拟合 Capacity sweet spot

mimo 的核心矛盾表述准确：8 prompt / 5777 帧 ≈ 37 万因子化 token——7B（RDT2-VQ/OpenVLA/FRoM-W1）必过拟合，135M 文本骨干（SmolVLA）又欠表达。甜点在 1.5–4B 冻结 VLM + 90M–200M 可训 head（现状）或 50–200M 从零小模型（路线 B）。约束是过拟合而非显存。

4.5 license 卫生普查 License hygiene

人形/动作生成领域 license 卫生极差：被核查的 16 个 motion 仓库里 8 个无 LICENSE 文件（UH-1、ScaMo、NORA、Wall-X 等 README 声称开源但法律上不可派生）。干净清单：StarVLA (MIT)、LeRobot/openpi/FAST/RDT2/T2M-GPT/1xgpt/MotionMillion/FRoM-W1 (Apache-2.0)、MoMask/MiniVLA/OpenVLA-OFT/MaskGIT-pytorch/TextOp (MIT)、GR00T (Apache-2.0 / NVIDIA Open Model License)。

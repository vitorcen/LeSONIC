GR00T-N1.7 × GEAR-SONIC — 三模型联合评审 + Roadmap

Three-model adversarial review of wsagi/GR00T-N1.7-G1-SONIC-BonesSeed · prompt 驱动全身动作的下一步

评审：Claude Opus 4.8 · OpenAI GPT-5.5 (codex) · Xiaomi MiMo-v2.5-pro  |  2026-06-08  |  架构侧 GR00T×SONIC-WBC 路线

TL;DR. 三个模型独立评审高度收敛：这是一个诚实、scope 写得到位的 proof-of-concept，但三座架构山不搬，数据再多也只是背更多轨迹——① 无记忆单帧策略做不了闭环时序动作、② 把离散 FSQ 码当连续回归、③ ego 相机看地面视觉近乎无效。最响亮的指标 MSE=0.0011 是 7 条轨迹无 held-out 的记忆误差，不是泛化。
Honest POC with accurate scope, but three architectural blockers (memoryless policy · continuous-regression of discrete FSQ codes · ground-facing ego cam) cap it at trajectory memorization. The headline MSE is train-set reconstruction, not generalization.

正序修复：先改 loss（最便宜、最高 ROI）→ 加 history/phase 治记忆 → 扩数据 → 闭环训练 → 实时 prompt。别先堆数据或堆步数。

0 · 评审方法 / Method

同一份事实简报喂给三个模型，各自独立产出批判 + roadmap，再交叉合并。三方独立命中同一问题视为强信号；单方独有视为补充视角；分歧处给出工程裁决。Same fact-brief → three independent critiques → cross-merge. Convergence = strong signal; singletons = supplementary; disagreements adjudicated.

4 · Roadmap — 让它真正「prompt 驱动更多动作」

泛化（新 prompt/新动作）+ 闭环稳 + 可实时下 prompt 三个目标，按 ROI 和依赖排序。Ordered by ROI and dependency, not by appeal.

⚠️ 2026-06-11 实施进展 / Implementation update — P0① 与 P1⑤ 已实测,结论修正 roadmap。 详见 StarVLA swap brainstorm §11。摘要:
  • P0① 改 loss = ✅ 成功(CE v1):per-dim 交叉熵把开环 MSE64 从 flow head 的 0.0374(背模板)降到 0.0174(比模板好 53%)—— roadmap 排序正确,这是最高 ROI 的一刀,已坐实。
  • P0① 的「升级版」联合分布 head = ❌ 负结果:MaskGIT masked CE(建 p(masked|revealed))在 5777 帧上 MSE64 0.0437,比常数模板还差 19%。小数据撑不起联合分布 + 迭代解码幅度塌缩(token std 跌到 GT 的 68%)。
  • P1⑤ obs/proprio history = ❌ 当前实现负结果:K=3 帧 history 经离散化塞 instruction 文本叠在 masked head 上,MSE64 0.0472(每窗都更差)。约 48% 历史 token 与当前帧同 bin = 半冗余稀释 cross-attn。不是 history 概念无用,是「文本通路 + 叠在塌缩 head 上」的工程选择错 —— history 若做,应走数值 state encoder 而非文本,且基于健康的 CE v1 而非 masked。
修正后的下一步(经 mimo 复审):先做两个 ~1 天消融排 confound,再决定是否转 P2。mimo 指出 masked 的负结果有两个未排除的更便宜解释: ① d_model 不公(CE v1=1024 vs masked=768,每位置表达力差 1.78×);② history 文本通路 ≠ history 概念(连续 state encoder 未测)。 幅度塌缩已验证是 confidence-ranking 揭示调度引起(随机揭示把 token std 从 68% 恢复到 83% GT),非 MaskGIT 范式死刑。 实验 A:masked + d_model=1024 + 随机揭示;实验 B:CE v1 + 数值 state encoder。两者都负 → 再投 5-10 天 P2 扩数据。 Per mimo's review: the masked-head loss has two cheaper unexcluded explanations (d_model 1024 vs 768; text-pathway ≠ history concept). The amplitude collapse is the confidence-reveal schedule (random reveal restores std to 83% GT), not a paradigm death. Run two ~1-day ablations before committing to P2 data scaling.

P0 · 修地基（1-2 周，零新数据，最高 ROI）

  1. 改 loss 为 per-dim 交叉熵（mimo 排序）：64 维 FSQ token → 64 路 16-class CE，或预测 pre-quant 连续 latent 再过 SONIC 自己的 FSQ quantizer。报 token accuracy / codebook perplexity / invalid-code rate。Single highest-ROI change; attacks #2 without new data.
  2. 建 held-out split：留 1 个动作完全不训（如 kick）+ 其余每条留后半段/某些 phase 做 val。没有 held-out 就没有科学。
  3. F32→bf16：重存 bf16 safetensors 或修 metadata，体积减半，说明推理显存/dtype。
  4. 页面去 oversell：标题/首屏标「memorized 7-trajectory POC」；MSE 旁注「train-set, no held-out」；每条 demo 标是否 bootstrap / 是否 RELAX；「不摔」在严格终止下重测后再写。

P1 · 治记忆（2-4 周，攻 #3 架构山）

  1. 加 obs history：DiT head 接 N=4-8 帧 obs 窗（拼 token 序列，不必上 RNN）。这是 kick/walk 自启的唯一正道。
  2. 显式 phase/progress 输入：proprio 里加 episode 内归一化时间戳，或 prompt 含 phase hint。训练时 random phase reset，让「从静止启动」「中途切 prompt」进训练分布。
  3. 去 bootstrap 拐杖：⑤⑥ 成立后 kick/walk 应能无 bootstrap 自启，这是 #3 是否真解的判据。

P2 · 扩数据（4-8 周，真泛化的燃料）

  1. 每动作 10-50 episode：变速度/幅度/起始相位/扰动。当前 1ep = 硬编码一条轨迹。
  2. held-out 新动作泛化测试：训 10 个 held-out 5 个，看 prompt 能否 zero-shot 选对——这才是 VLA 卖点。
  3. 数据增强：token 序列 time-warp（±20% 速度）/ phase shift / magnitude scale。
  4. prompt paraphrase：同义改写测 token 是否稳定指向正确动作（数据多样性 ROI：prompt 侧 > DR > multi-clip，见项目记忆）。

P3 · 闭环 + 解耦（与 P2 并行）

  1. 闭环训练：别只 teacher-forcing MSE。scheduled sampling / DAgger / noise injection，让模型见到自己的偏差（codex）。
  2. ablation 矩阵解耦 VLA vs WBC 贡献：WBC-replay / mean-token / prompt-shuffle / proprio-only / vision-off。证明视觉/prompt 真有贡献（codex），也据此决定 ego cam 去留。
  3. 严格终止下重测：关掉 RELAX，统计 fall rate / 存活时长 / phase error / 成功率（统一定义 + 多 seed + bootstrap/no-bootstrap 分开算）。

P4 · 实时 prompt（长期，依赖泛化先成立）

  1. prompt transition 数据：录「dance→kick→walk」切换段，定义 latency budget，每 N step 更新语言条件。
  2. live get_action 活循环（ZMQ 已通），交互式实时下 prompt。
  3. 组合 prompt：「squat while waving left hand」测 token 空间组合泛化——若平滑，才算真 VLA。
正序提醒 / Ordering. 三方一致：先 loss，再 history，然后数据。P0 的 held-out 是整条路的 go/no-go——若 held-out phase 都崩，P2/P3/P4 全是给查找表加表项，不值得投入。P0 held-out is the go/no-go gate; if it fails, everything downstream just enlarges the lookup table.

5 · 该页面立刻可做的诚实化（不需训练）/ Immediate honesty fixes

  • 标题/首屏：「POC: 7-trajectory memorization + SONIC WBC base」，把通用能力的暗示拿掉。
  • MSE 0.0011 改措辞为「train-set token reconstruction error (no held-out)」，并列出 per-motion-mean=0.039 是弱基线的事实。
  • 每条 demo 视频标注：bootstrap? RELAX(终止放宽)? kick/walk 明确写「需 bootstrap」。
  • 「pelvis 稳定不摔」：注明是否在 RELAX 下测；若是，撤下或严格终止重测后再写。
  • 修 F32→bf16，或在 card 注明 dtype/推理显存。
  • 补最小复现包：WBC ckpt 版本/commit、env lockfile、ZMQ wire schema、token 抽取脚本、eval/inject 命令、bootstrap 参数。当前更像研究草图而非可复现发布。

6 · 公允评价 / Fair verdict

这套工作架构侧的去风险目标已达成：prompt→GR00T→token→SONIC decode→G1 端到端内存跑通（无 C++/ZMQ 阻塞主流程），注入接口验证 OK，模型在训练集内确实学会 (ego,prompt)→token 映射。页面的 limitations/scope 写得诚实，没有冒充通用 VLA。

但把它当「成果」发布的成色，取决于能不能过 P0 的 held-out 与严格终止重测。当前最响亮的两个数字（MSE 0.0011、不摔 0.79m）一个是记忆误差、一个疑似 confound——这两条不澄清，页面的科学可信度会被懂行的人一眼看穿。先把页面诚实化（半天），再决定是否投 P0-P4。
